Books API - documentation

/BOOKS

All

GET /books get all books

Search by

GET
/books/search?s=key-word
Get books by searching a full/partial name or info
  • you can add pagenation:
    • /books/search?s=key-word&page=page-number&perPage=books-perPage
    {defualt page = 1 , defualt perPage = 10}
    GET
    /books/category/catName
    Get books by searching a category. Valid options: Romance /Adventure /Science Fiction /Fantacy /Drama /Mystery novel
  • you can add pagenation:
    • /books/category/catName?page=page-number&perPage=books-perPage
    {defualt page = 1 , defualt perPage = 10}
    GET
    /books/prices?min=minPrice&max=maxPrice
    Get books by choosing thier maximum and/or minimum price
  • you can add pagenation:
    • /books/prices?min=minPrice&max=maxPrice&page= page-number&perPage = books-perPage
    {defualt page = 1 , defualt perPage = 10}
    GET
    /books/single/idBook
    Get one book by its id
  • you can add pagenation:
    • /books/prices?min=minPrice&max=maxPrice&page = page-number&perPage=books-perPage
    {defualt page = 1 , defualt perPage = 10}

    Add book

    POST
    /books
    Post a new book. Only registered users can add a new book (send a valid token for identification)
    Your request must include the following properties:
    1. name
    2. info
    3. category
    4. price
    5. img_url (optional)

    Edit book

    PUT
    /books/ EditId
    Edit a book by its id. Only the user that posted the book can edit it OR an admin.(send a valid token for identification)
    Your request must include the following properties:
    1. name
    2. info
    3. category
    4. price
    5. img_url (optional)

    Delete book

    DELETE
    /books/ Del-Id
    Delete an existed book by its id. Only the user that posted the book can delete it OR an admin.(send a valid token for identification)

    /USERS

    Sign Up

    POST
    /users/signUp
    Signing Up
    Your request must include the following properties:
    1. name
    2. email(must be unique)
    3. password
    4. role(optional)

    Login

    POST
    /users/login
    Login(if the process has succeeded, you will receive a token valid for 60 minutes)
    Your request must include the following properties:
    1. email
    2. password

    User information

    GET
    /users/userInfo
    Get user information. Every user can only see his own information.(send a valid token for identification)
